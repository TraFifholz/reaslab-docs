# Git 同步集成

为保持团队协作中的外部版本追踪一致性，ReasLab 内有独立的推（Push）与拉（Pull）机制连接远端 Git。本指南包含操作总览与远程连接方法说明。

## Git 线上联通机制

无论是首次空项目设立外部节点，或基于第三方工程部署代码，ReasLab 都赋予您极易部署的操作集。
*注意：对于直接使用 GitHub OAuth2 授权并在本地衍生项目的情景下，您的账号拥有即刻且自动化的凭据而不需要进行手工连通认证。*

### 唤醒同步面板

绝大部分的集成互动放置在中央操作区：
1. 单击顶部集成栏的 **Menu** 图标。
2. 转场至 “Actions” 。
3. 选择 **Sync Push To Remote 这里操作推送**，或者是 **Sync Pull From Remote 拉取更新**。

#### Sync Push To Remote (推至云端)
- 将此一阶段的产出数据向建立好路由配置的 Git 进行更新递交。
- 全程具有防阻塞的后台传输提示设计。
- 操作成功后会显露校验绿条。
- 基础前提条件是 URL 正确无误且无私有云的密钥门槛阻挠。

#### Sync Pull From Remote (吸纳云端)
- 在保留现存状态基调下合并吸收最新版远端工程变更物。
- 同理赋予可视化流转预告。
- 合并并写入您的环境库。

## 设置中心

配置连接需要在您专门的 Project Settings 里发起：

### Git 远程端地址
设立一个基于 HTTPS 开拓的通用 Remote URL 目标载体库。

### Git 加密权限组
由于权限保护机制往往封锁未经过审核的分支请求，请在此：
- 配置您的私域仓库个人安全密钥访问凭证 (Personal Access Tokens)。
- Token 等机密资料均会被极小范围存储而不面临安全逸散与信息流监控威胁。

## 由现存 Git 转变为 IDE 项目库
将既有财产纳入正式流程的方案非常多元：
1. **GitHub 生态植入**：只要账户勾稽完毕，您可以直通式地抓取列表而免于手工填充资料。
2. **多端接口映射**：任何规范化的代码端公开 URL 亦能轻松拷贝载录。
3. 现存系统将锚定于获取其设置为主分支 (Default Branch) 的资源树作为基础母版。

## 当前的集成特性限界

受限于我们的交互逻辑重点构建工作流在 AI 理学工程领域而非单纯化源码仓库：
### 目前全支持部分
- ✅ 向运端同步 Push 改动
- ✅ 自远端合并 Pull 到现实例子
- ✅ Remote 和 Tokens 的定义映射
- ✅ 从 GitHub 等端口克隆初始生命周期
- ✅ 主账号生态无缝链接

### 尚未开发现有功能缺环
- ❌ 底层繁杂的各类 Branch checkout 和管理派生操作
- ❌ 直接基于我们前端审视巨量的 Commit 时间回溯轴
- ❌ 手动且图形化的超高难度合并冲突 (Conflict Resolution) 排解视窗操作
- ❌ 各类非线性撤回、再置入工作（Rebase，Stash 或 Cherry-Pick）
- ❌ 原生的 Git Diff 代码比对展现
- ❌ 特定的自适应定制 Commit Message 填装
如果您非常需要超复杂的源码基线干预与版本审计系统，不妨在外部调用对应的传统 Git 图形化工具与我们并列工作以互补所需。
