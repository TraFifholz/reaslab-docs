# Lean 工具链支撑

ReasLab 内嵌了经过特殊调优的浏览器级 Lean 语言服务器，无需任何终端配置即可获得极速反馈。本指南重点介绍 Lean 4 的交互式定理证明流程：包含 Tactics（战术应用）、Goals（目标判定）、Hypotheses（已知预设）、Infoview，以及 AI 语义搜索等核心功能。

## 语法高亮与编辑器偏好

![语法高亮示例](/images/lean-syntax.png)

- Lean 代码以符合其语言特性的高光色标进行深度渲染，区分战术、项、类型、导入等不同语法成分，一目了然。
- 在编辑器设置面板中调整主题、字号、行号显示、换行方式和光标样式。
- **括号匹配**与**自动闭合**（`()`、`{}`、`[]`、`«»`）让您在嵌套极深的证明项中轻松导航。

## 悬停与定义跳转

![悬停提示](/images/hover-tooltip.png)

- 将鼠标光标悬停在任意 Lean 代码位置，即可内联查看语句的类型、声明和文档注释。
- 使用**转至定义**（`F12` 或右键菜单）或**转至类型定义**可跳转到符号的声明位置。
- 来自依赖项（标准库、MathLib 或任何项目依赖）的定义会在只读视图中打开，不会意外修改主文件。
- **速览定义**（`Alt + F12`）以内联弹窗展示定义内容，无需离开当前光标位置。

## 诊断消息

![行内诊断](/images/inline-diagnostics.png)

- 编辑区装配了全生命周期的行内诊断：错误为红色波浪线，警告为黄色，信息为蓝色，并在 gutter（区距）中显示图标。
- **Infoview**（信息视窗）与 Lean 诊断保持实时同步，持续更新，无需人工刷新。
- 诊断信息以流式方式实时推送，无需等待「构建步骤」完成。

## Infoview 工作流

![Lean Infoview](/images/lean-infoview.png)

- 为 `.lean` 文件自动在右侧面板开启 **Infoview**，实时展示当前证明状态。
- **目标面板**（Goals）：显示当前证明目标与子目标，每个目标编号显示其类型和预期结论。
- **假设面板**（Hypotheses）：列出所有当前可用的假设（变量、参数及已证明的引理）及其类型。
- 使用 Infoview 的**刷新按钮**（或 `Cmd/Ctrl + R`）重启分析——网络波动或工具链状态陈旧时尤其有用。
- **战术状态差异**：每次编辑后，Infoview 可高亮显示证明状态的变化，帮助您理解某条战术成功或失败的原因。

## 战术 (Tactics) 与证明目标 (Goals)

通过叠加不同的战术 (Tactic) 向目标稳步推进。以下是常用战术模式：

| 战术 | 用途 |
|------|------|
| `rfl` | 证明形如 `x = x` 的目标 |
| `simp [*]` | 利用引理和假设化简目标 |
| `exact term` | 提供具体项作为证明 |
| `have h : T := proof` | 引入中间引理 |
| `calc` | 在结构化证明块中链接等式或不等式 |

在 Infoview 中追踪您的**目标 (Goals)** 和**假设 (Hypotheses)**，理解每步战术前的证明状态，再决定下一步如何攻坚。

## Unicode 符号全矩阵

![符号面板](/images/symbol-palette.png)

- 输入反斜杠前缀可快速插入常用 Unicode 符号：`\to` → `→`、`\forall` → `∀`、`\exists` → `∃`、`\lam` → `λ`、`\le` → `≤` 等。
- 通过编辑器工具栏或 `Cmd/Ctrl + Shift + Space` 调用**全局符号面板**，按类别浏览和搜索所有数学符号，并实时预览字符形状。
- 在 Lean 文件中，符号面板会根据上下文优先显示数学和逻辑类 Unicode 符号。

## 语义搜索

![语义搜索](/images/search-semantic.png)

除传统全文搜索外，ReasLab 还提供 **AI 驱动的语义搜索**，能够理解数学概念而非仅匹配关键词：

- 在左侧边栏切换到 **语义搜索（Semantic）** 面板。
- 使用自然语言描述您要找的内容，例如「矩阵分解方法」或「凸优化对偶性」。
- 搜索引擎跨越代码结构和数学语义进行检索，即使代码中未出现确切关键词，也能找到相关的 Lean 定义和引理。
- 结果按语义相关性排序，并高亮显示匹配的代码片段。

## Lean 工具链管理

![新建项目 — 工具链](/images/new-project-toolchain.png)

在开始编辑前，选择与项目匹配的 Lean 版本。ReasLab 始终显示当前激活的工具链版本，让您清楚了解当前环境可用哪些编译器特性。

### 新建项目时选择版本

- 在**新建项目**流程中，从下拉菜单中选择 Lean 工具链版本（如 stable、nightly 或指定版本）。
- IDE 自动配置正确的工具链版本及关联库。
- 从 GitHub 导入已有仓库时，若仓库包含 `lean-toolchain` 文件，ReasLab 会严格遵循该配置，不随意升降版本。

### 确认当前工具链

- 查看编辑器底部**状态栏**中的当前模式和 Lean 版本信息。
- 若 Infoview 显示状态陈旧，使用刷新按钮以当前工具链重启分析。
