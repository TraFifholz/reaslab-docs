# 项目与导入

本指南介绍了如何创建新工作区、导入现有仓库，以及通过链接加入共享项目。ReasLab 支持为不同使用场景量身定制的多种专业项目模板。

## 创建新项目

ReasLab 提供了多种启动方式，无论您是想从零开始还是使用预配置的环境，都能找到合适的入口。

### 1. 创建空白项目 (New Project)
适用于通用的科学写作或自定义工程结构。通常点击 Dashboard 侧栏的 **New Project**。
![创建空白项目](../../reference/images/paper_image1.png)
- 输入唯一的 **Project Name** 项目名称。
- 如果您的项目涉及 Lean，请通过下拉框选择对应的 **Lean Toolchain 编译版本**。
- 点击 **Create** 即可拉起工作区。

### 2. 通过数学建模模板创建
专为竞赛参与者设计，模板自带题目背景和报告结构。
![数学建模模板](../../reference/images/math_image1.png)
- 在 Templates 区域选择 **Math Modeling Contests** 分类。
- 选择具体的竞赛题目以查看详情。
- 点击 **Use Template** 实例化该项目。

### 3. 通过优化建模模板创建
快速引导带有数学模型和求解器配置的优化问题。
![优化建模模板](../../reference/images/opt_image1.png)
- 导航至 **Optimization Modeling** 选区。
- 在分类树中浏览各种复杂的优化模型子类。
- 对选中的模型点击 **Use Template**。

### 4. 通过 Lean (定理证明) 模板创建
使用预配置的 Lean 4 环境启动形式化验证项目。
![定理证明模板](../../reference/images/lean_image2.png)
- 进入 **Theorem Proving** 分类。
- 选择符合您证明需求的工程模板。
- 点击 **Use Template** 开始。

## 从 Git 导入

![Import Git](/images/import-git.png)

1. 在导入页面粘贴任何 Git URL 以进行手动集成。
2. 如果您使用 GitHub 登录，您的代码库将自动出现；否则，请在“帐户设置”中链接 GitHub 帐户。

## 通过分享链接加入

- 如果您拥有其他成员提供的高权限分享链接，直接在浏览器中打开链接并在登录账号后，即可按授予的权限（如查看、编辑、管理等）加入并在多人协作模式下共同工作。
