# Files & History

Manage project structure and review past work without leaving the IDE. This guide explains how the file tree and history viewer operate together.

## File tree operations

- Create files or folders from the left sidebar, rename items in place, and delete multiple selections at once.
- Drag files to move them within the project hierarchy.

## Upload files

![Upload Files](/images/upload-files.png)

- Drag and drop files or folders onto the file tree to upload into the selected folder.
- Or use the Upload action in the Files panel to pick items from your device.

## Project history

![Project History](/images/project-history.png)

- Open the **History** view from the top bar or visit `/projects/<projectUuid>/history`.
- Browse snapshots to see how the project evolved over time and inspect per-file diffs.
- Use **Checkout This Snapshot** to restore a previous state when you need to roll back changes.

Background autosave keeps snapshots current so you can recover work even if collaborators disconnect.

## Rename project

- In the Files tree, double‑click the project root name to rename it in place.

![Rename in Dashboard](/images/rename-project.png)

- Or from the dashboard, open All Projects and rename the project directly from the list.
